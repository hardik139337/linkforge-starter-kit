## Reddit — r/nextjs

### Title
Built a Next.js 16 SaaS boilerplate with App Router, Prisma 7, NextAuth v5, and Stripe — all working out of the box

### Body
Hey r/nextjs!

I built a production-ready SaaS boilerplate that handles all the infrastructure stuff we all build from scratch. Figured some of you might find it useful.

**Stack:** Next.js 16 (App Router + Turbopack) · Prisma 7 · NextAuth v5 beta · Stripe · Tailwind v4

**What's wired up:**
- Auth: NextAuth v5 with credentials provider, JWT strategy, cookie-based middleware
- Payments: Stripe checkout sessions, webhook handlers for subscription lifecycle
- Database: Prisma 7 with BetterSqlite3 adapter (swap to PostgreSQL with one env var)
- Dashboard: Full CRUD for links, drag-to-reorder, phone mockup preview
- Public profiles: Slug-based URLs with 5 themes
- Analytics: Page views, link clicks, aggregated dashboard
- Landing page: Hero, features grid, pricing table, FAQ

Had to work through some fun edge cases:
- Prisma 7 dropped `url` from datasource — uses driver adapters now
- NextAuth v5 beta has different API from v4 (no `pages` config, uses `auth()` function)
- Edge Runtime can't load Prisma — had to use cookie-based middleware instead

All the code is on GitHub: https://github.com/hardik139337/linkforge-starter-kit

Happy to answer questions about the stack choices or edge cases!
